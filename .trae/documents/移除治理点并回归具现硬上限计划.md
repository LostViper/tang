# 移除治理点并回归具现硬上限计划

## Summary

本次目标是把最近补入的“治理点”体系整体撤回，恢复到更克制的具现设定：蓝星物资具现继续只受硬边界约束，核心口径维持“每七天一次，单次原则上不超过 50 公斤或 0.2 立方米，且仅限非生命、结构清晰、现实可承接对象”。

这次不是只改一处台词，而是要把已经写入设定文和正文中的治理点痕迹一起清理干净，避免口径互相打架。

### 4. 把“可兑换物资”改写为“可具现物资分级”

- **文件**：优先落在重写后的 `16-...md`
- **做法**：
  - 去掉“兑换资格”“治理点够不够”这类表达
  - 改成按现实用途分层：
    - 前期低风险剧情物资
    - 民生与医药救急物资
    - 工艺点火物资
    - 高敏感战略物资
    - 禁止或长期冻结项
  - 每层都只说明：
    - 可具现什么
    - 适合什么剧情节点
    - 为什么不能长期代供
- **原因**：
  - 你现在要保留的是“物资列表”，不是“兑换经济系统”

### 5. 统一后续叙事口径

- **影响范围**：
  - 后续凡涉及“能不能换、能不能掏、怎么具现”的写法
- **要统一的口径**：
  - 蓝星能具现的依据，不是治理点
  - 而是：
    - 当前是否处于可具现周期
    - 单次总量是否超限
    - 对象是否非生命且边界清晰
    - 大唐现实是否接得住
    - 主角是否讲得圆来路
- **原因**：
  - 这是去掉治理点之后，整套设定还能稳住的关键

## Assumptions & Decisions

- **决策 1**：治理点体系整体撤回，不保留任何残余接口。
- **决策 2**：50 公斤 / 0.2 立方米 + 每七天一次，继续作为唯一核心硬边界。
- **决策 3**：保留“物资清单”这个需求，但把“兑换清单”改造成“可具现物资分级清单”。
- **决策 4**：不把蓝星重新写成普通系统商城；只是回到“有限具现样品/补丁”的旧路线。
