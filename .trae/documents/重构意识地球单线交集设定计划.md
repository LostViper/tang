# 重构意识地球单线交集设定计划

## 背景判断

当前 `01-核心设定` 目录仍保留了“现代地球复制体整体进入贞观时空”的旧底盘，而你现在要改成：

- 不再有“现代地球复制体进入贞观时空”这件事
- 与贞观时空发生交集的只有主角关联的“意识地球”
- 主角的超常性来自意识地球的认知统摄、内部绝对服从样本与少量物质具现

这意味着当前目录里不只是几句措辞要改，而是底层世界模型要从“双世界并存”切换成“单现实世界 + 主角意识接口”。

## 目标

- 把 `01-核心设定` 目录从“现代地球复制体降临”完整改写为“只有意识地球与贞观世界发生交集”
- 保留主角的强势感，但去掉会误导成现实双世界并存的表述
- 让各专题继续能支撑剧情，同时与新总纲完全一致

## 实施步骤

### 第一步：重写总纲的底层世界模型

目标文件：

- `01-核心设定/00-核心设定总纲.md`

具体处理：

- 删除或改写所有“现代地球复制体进入贞观初年”“复制体保留现代地球陆地人口国家”“现代地球与大唐并存”相关表述
- 将总前提改为“现实中只有贞观时空，大唐世界是唯一现实场域”
- 明确意识地球不是现实地球的映射副本降临，而是主角意识内的完整文明样本库、组织协同场与权限接口
- 重写“初始状态”，改成主角本人与贞观世界接触后的起始局面，而不是全球同步进入新环境
- 重写“总优势”与“总限制”，把优势集中到主角的知识整合、推演能力、少量具现和组织统摄；把限制集中到现实只有一人落地、需要借壳现实资源、无法直接搬运现代人口政权工业体系
- 重写“必答问题”，从“两个世界如何接触”改成“主角如何用意识地球影响贞观世界”

### 第二步：按专题逐篇替换旧世界观前提

目标文件：

- `01-核心设定/README.md`
- `01-核心设定/01-地理与空间格局.md`
- `01-核心设定/02-人口与制度接驳.md`
- `01-核心设定/03-物资储备与工业底盘.md`
- `01-核心设定/04-技术传播与能力边界.md`
- `01-核心设定/05-政治重构与权力冲击.md`
- `01-核心设定/06-文化碰撞与认知断层.md`
- `01-核心设定/07-社会秩序与连锁冲击.md`

具体处理：

- README 中把目录说明改成“主角以意识地球为接口切入贞观世界”的资料区
- 地理专题改成“现实地理只存在贞观世界”，重点讨论主角如何借意识地球进行地图推演、路线判断、资源节点选择，而不再讨论现代全球地图降临后的空间关系
- 人口与制度专题改成“贞观人口与制度如何被主角带来的现代治理认知冲击”，而不是现代全球人口整体迁入后的身份归属问题
- 物资与工业专题改成“主角如何依靠少量具现撬动贞观现实中的工坊、冶炼、药材、运输和组织能力”，而不是现代工业底盘如何维持
- 技术专题改成“意识地球提供知识与样品，贞观现实负责学习、试错、复做和扩散”
- 政治专题改成“主角如何在大唐权力结构中被理解、利用、防范或神化”，而不是现代国家体系与唐廷并列博弈
- 文化与社会专题改成“主角带来的现代认知和奇迹化行为如何冲击贞观社会”，去掉现代大众社会同步存在的前提

### 第三步：统一术语与叙事边界

目标范围：

- `01-核心设定` 目录全部文档

具体处理：

- 统一保留“意识地球”“最高统摄权”“最高纲领”“少量具现”“现实贞观世界”这组核心术语
- 删除或替换“现代地球复制体”“现代国家并存”“全球人口同步进入”“现代社会内部国际结构”这类旧设定词
- 明确任何专题都不能再默认存在一个现实层面的现代全球系统
- 保持主角外挂边界不变：只能认知统合、少量具现、不能无限供给、不能直接搬运现代社会本体

### 第四步：补强新的剧情支点

目标范围：

- `00-核心设定总纲.md` 及 01-07 各专题

具体处理：

- 在各专题补出“主角一个人如何借意识地球撬动大唐现实”的核心问题
- 强化“主角对现实的影响需要通过人、组织、制度、工艺、资源和威望中介实现”
- 强化“主角越像神迹源头，越容易引发政治警惕、文化误解和社会过载期待”
- 让每篇文档都能直接转化成剧情冲突，而不再依赖“全球现代体系已落地”的旧便利

### 第五步：做一致性核对

核对重点：

- 是否还残留“现代地球复制体进入贞观时空”的旧句子
- 是否还有把现代全球人口、国家、工业体系当作现实现成资源的表述
- 是否所有专题都已切换到“主角接口型穿越”视角
- 是否保住主角强度，同时避免设定失衡

## 实施后的预期结果

- 核心设定会从“大型双世界碰撞”转成“单人携带意识文明接口切入大唐”
- 故事重心会更集中在主角如何影响李世民时代，而不是全球级并存秩序
- 设定会更利于人物戏、权谋戏、技术传播戏和奇迹政治学

## 执行顺序建议

1. 先改 `00-核心设定总纲.md`
2. 再改 `README.md`
3. 然后按 `01 → 07` 顺序逐篇改专题
4. 最后全目录检索旧术语并做一致性校正
